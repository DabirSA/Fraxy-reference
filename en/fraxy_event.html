Condition Checks & Settings
Condition Check Types
- [AND]
The event will execute if all of its conditions are TRUE.

- [OR]
The event will execute if at least one of its conditions is TRUE.

Extra Settings
- Don't delete this event after execution
If this is checked, the event will execute once on every frame that its conditions are met.

- Deletes if condition is disagreement
If this is checked, the event will be deleted if its conditions are ever not met.

Condition Operators
= (equal)
A = B
If A and B are equal, this condition is TRUE.
If they are not equal, it is FALSE.

! (not equal)
A ! B
If A and B are not equal, this condition is TRUE.
If they are equal, it is FALSE.

> (greater than)
A > B
If A is greater than B, this condition is TRUE.
If A is less than or equal to B, this condition is FALSE.

< (less than)
A < B
If A is less than B, this condition is TRUE.
If A is greater than or equal to than B, this condition is FALSE.

>= (greater equal)
A >= B
If A is greater than or equal to B, this condition is TRUE.
If A is less than B, this condition is FALSE.

<= (less equal)
A <= B
If A is less than or equal to B, this condition is TRUE.
If A is greater than B, this condition is FALSE.

& (and)
A & B
Compares the binary bit patterns of A and B.
If at least one of them matches, this condition is TRUE.
If none of them match, this condition is FALSE.
Fraxy has a built-in bit calculator which will help you to use this operator.
Examples:
13(1,4,8) & 4(4): TRUE
13(1,4,8) & 2(2): FALSE
13(1,4,8) & 10(2,8): TRUE
13(1,4,8) & 12(4,8): TRUE
13(1,4,8) & 18(2,16): FALSE

&! (and + not)
A &! B
Compares the binary bit patterns of A and B.
If none of them match, this condition is TRUE.
If even one of them matches, this condition is FALSE.
Fraxy has a built-in bit calculator which will help you to use this operator.
Examples:
13(1,4,8) &! 4(4): FALSE
13(1,4,8) &! 2(2): TRUE
13(1,4,8) &! 10(2,8): FALSE
13(1,4,8) &! 12(4,8): FALSE
13(1,4,8) &! 18(2,16): TRUE

&= (and + equal)
A &= B
Compares the binary bit patterns of A and B.
If all of B's bits are found in A, this condition is TRUE.
If even one of them is not, this condition is FALSE.
Fraxy has a built-in bit calculator which will help you to use this operator.
Examples:
13(1,4,8) &= 4(4): TRUE
13(1,4,8) &= 2(2): FALSE
13(1,4,8) &= 10(2,8): FALSE
13(1,4,8) &= 12(4,8): TRUE
13(1,4,8) &= 18(2,16): FALSE
Condition Types
Player Status
- P_Control status: 0~1
If the player has control of their ship, this is 1. If they do not, this is 0.
Returns 0 during player destruction and while they're returning to the field after selecting Continue.

- P_Destroy value: 0~
Normally returns 0.
Returns 1 at the moment the player is destroyed, then increaes by 1 during each frame of the player's destruction animation.

- P_Stun value: 0~
Normally returns 0.
Returns greater than 0 if the player is stunned.

- P_Energy: 0~100%
Returns the player's current energy level as a value between 0 and 100.

- P_Stock: 0~100%
Returns the player's current energy stock as a value between 0 and 100.

- P_Using equip: 0~2
Normally returns 0.
Returns 1 when the player has just pressed the button to use a weapon or support item.
Returns 2 if a weapon or support item is currently in use.

-P_Using weapon: 0~2
Normally returns 0.
Returns 1 when the player has just pressed the button to use a weapon.
Returns 2 if a weapon is currently in use.

- P_Using support: 0~2
Normally returns 0.
Returns 1 when the player has just pressed the button to use a support item.
Returns 2 if a support item is currently in use.

- P_Boost button: 0~
Returns 0 if the boost button is not being held.
Returns 1 on the frame that the boost button is pressed, then increases by 1 for each frame that it is held.

- P_Boost: 0~
Returns 0 if the player is not moving while boosting.
Returns 1 on the frame that the player starts moving while boosting, then increases by 1 for each frame that they continue moving while boosting.

- P_Move: 0~15
Returns 0 if the player is not inputting movement.
If the player is inputting movement, acts as a bitfield with the following bits for each direction:
Left: &1
Up: &2
Right: &4
Down: &8

- P_Extra button: 0~
Returns 0 if the player is not pressing the Extra button.
Returns 1 on the frame that the Extra button is pressed, then increases by 1 for each frame that it is held.

- P_Slot current: 1~5
Returns which of the player's five equipment slots they currently have selected.

- P_Slot1~5 use: 0~2
Normally returns 0.
Returns 1 when the player has just pressed the button to use this slot.
Returns 2 if the weapon or item in this slot is currently in use.

- P_Slot1~5 type: 0~
Returns 0 if this slot is empty.
If this slot is not empty, returns the ID of the weapon or item it contains.

- P_Slot1~5 level: 1~5
Returns the current base level of the weapon or item in this slot.
Does not include any modifiers from Soll Activator or Over Soll Power.

- P_Slot1~5 ExLevel: 1~5
Returns the current level of the weapon or item in this slot.
Does include any modifiers from Soll Activator and Over Soll Power.

- P_Slot1~5 status: 0~100
Returns the status value reported by the weapon or item in this slot.
For example, Charge Blasters report their current charge level as a value between 0 and 100.

Distance and Angle
- Distance (Player): 0~
Returns to the straight-line distance between this part and the player.

- Distance (Pointer): -1 or 0~
Returns the straight-line distance between this part and its pointer part.
Returns -1 if a pointer part could not be found.

- Distance (Pair): -1 or 0~
Returns the straight-line distance between this part and its paired part.
Returns -1 if a paired part could not be found.

- Angle to Pair/Pointer: -2048~2047, 0~2048 (Abs)
Returns the angle between the target part's position and the forward direction of this part.
Returns 0 if the target part is directly in front of this part.
Returns 1~2047 if the target is on the right, -2048~-1 if the target is on the left.

The (Abs) version of this condition returns the absolute value of the angle, in a range of 0~2048.

- angle to P_Pos: -2048~2047, 0~2048 (Abs)
Returns the angle between the player's position and the forward direction of this part.
Returns 0 if the player is directly in front of this part.
Returns 1~2047 if the player is on the right, -2048~-1 if the player is on the left.

The (Abs) version of this condition returns the absolute value of the angle, in a range of 0~2048.

Movement/Attack Actions
- MovAct/AtkAct(StopLoop)
Returns TRUE in cases where the part's movement or attack action loops into itself infinitely.

For example, Power Laser's attack action [8] jumps back into attack action [8]. AtkAct(StopLoop) returns true in this case.

Miscellaneous
- Rand()
Returns a value between 0 and n-1, where n is the number you input.
If you enter a negative number, returns a value between n+1 and 0.

For example, Rand(40) returns a number between 0 and 39, and Rand(-40) returns a number between -39 and 0.

- Previous Event Status: -1~1
This is a special condition which returns the execution status of the event immediately above this event in the part's event list.
Returns 0 if the event above has not executed.
Returns 1 if the event above has executed.
Returns -1 if this event is at the top of the event list.
If there were other events above this event, but they have been deleted, returns -1.

- Part Exists
Checks whether the specified part exists.
Returns 1 if it exists, 0 otherwise.
Change Connect Type
Connect Types
- (0) Fix (Available values: 0~0)
Part is rigidly attached to its parent, if it has one.

- (1) Tail (Available values: 1~100)
Part moves as if being loosely pulled by its parent.
The part's center distance is used in its calculation, so if center distance is set to 0, there will be no noticeable sway.
The value you enter detemines how softly the part sways. The lower the value, the softer the motion. (Soft) 1 ----- 100 (Hard)

- (2) Player (Available values: 1~100)
Part's position and angle are based on those of the player.
The value you enter detemines how strongly the part follows the player. The larger the value, the more weakly it follows.
(Strongly) 1 ----- 100 (Weakly)

- (3) Player (Pos only) (Available values: 1~100)
Part's position is based on that of the player.
The value you enter detemines how strongly the part follows the player. The larger the value, the more weakly it follows.
(Strongly) 1 ----- 100 (Weakly)

- (4) Alone (Available values: 1~100)
Part detaches from its parent and remains at its current position and angle.
Value has no significance, but set it to 1 in case it gains some later.

- (5) Re-fixing (Available values: 1~100)
The part returns to its position relative to its parent over the course of 1~100 frames.
After returning to its fixed position, it acts as if it has connect type (0). The value you enter detemines how many frames the part takes to return to its fixed position.

- (6) Half-fixing (Available values: 1~100)
The part returns to its position relative to its parent, but loosely.
The value you enter detemines how strongly the part returns to its position. The larger the value, the more weakly it follows.
(Strongly) 1 ----- 100 (Weakly)

- (7) Tail Re-fixing (Available values: 1~100)
The part starts out connected like connection type (1), then switches to type (0). The value you enter determines how long the part takes to switch connection types.

- (8) Player (Axis) (Available values: 1~100)
The part tracks the player's position, but only moves along an axis based on its parent's position and facing.
The value you enter detemines how strongly the part follows the player and its axis. The larger the value, the more weakly it follows.
(Strongly) 1 ----- 100 (Weakly)

- (9) Midpoint (Available values: 1~100)
The part takes a position halfway between its parent and its pointer part.
Its angle is based on the angle between its parent's position and its pointer part's position.
The value you enter detemines how strongly the part tracks its pointer. The larger the value, the more weakly it tracks.
(Strongly) 1 ----- 100 (Weakly)

- (10) Pointer (Available values: 1~100)
The part's position and angle are determined by its pointer part.
The value you enter detemines how strongly the part tracks its pointer. The larger the value, the more weakly it tracks.
(Strongly) 1 ----- 100 (Weakly)
Part Status
Part Status
- Change damage rate
Damage taken rate: ~0~ (default: 100%)

At 100, the part takes 100% damage from all sources.
At 50, the part takes half damage. At 0, it takes no damage.
If negative, the part gains health instead of taking damage.

- Change bullet power
Bullet Power: 0~n (default: 10000)

Changes the power of enemy bullets generated by this part.
At the default of 10000, bullets have 100% of their normal power.
Drawing & Display Flags
Health Bar
- Main health bar
OFF: Target part does not use the main health bar.
Use (if not used): Target part uses the main health bar if no other parts are already using it.
Use (by force): Forces target part's use of the main health bar even if other parts are already using it.

- Sub health bar
OFF: Target part does not use a sub health bar.
ON: Target part uses a sub health bar.

Custom bar
- Custom bar
Lets you display part-specific parameters, variables and global variables in the form of a bar.
The custom bar is a sub health bar under the hood, so you can only have one of them attached to each part.

Value A is the value you want the custom bar to track.
Set it to OFF to remove the custom bar.

Value B is the maximum value of the bar.
If Value B is 1 or less, the custom bar will be removed.

- Custom bar settings
These are various options which control the appearance of the custom bar.
For Mode, combine the flags below to control its behaviour in detail.

&1: Deletes bar when the value is 0 or below
&2: Displays an afterimage which decays slowly when the bar decreases
&4: Afterimage doesn't decay until part is not being hit
&8: Perform score-related processing (Unclear what this means)
&16: Draws a glow effect when the bar's value is maximum
&32: Always draws a glow effect
&64: Draws a flash effect when the bar's value reaches maximum
&128: Bar is vertical
&256: Refer to pair part 1
&512: Refer to pair part 2
&1024: Refer to pair part 3
&2048: Refer to pair part 4

For example, if you want to create a custom bar that's identical to a sub health bar, set Value A to Durability, Value B to Durability (Max), and Mode to 15 (1+2+4+8).

Pair part flags change the source of values A and B.

- Change parts draw order shift value
Parts draw order shift value: ~0~

Shifts the part's draw order by the given amount.
For example, if the part's draw order is (5) and you specify 1 for the shift value, the part will be drawn on layer (6).
The shifted draw order will not go beyond the normal range of draw order values.
The original draw order will be kept, allowing you to easily handle multiple draw orders.

- Change effect draw order shift value
Effect draw order shift value: ~0~

Shifts the effect's draw order by the given amount.
For example, if the effect's draw order is (5) and you specify 1 for the shift value, the effect will be drawn on layer (6).
The shifted draw order will not go beyond the normal range of draw order values.
The original draw order will be kept, allowing you to easily handle multiple draw orders.

- Change shot effect type
Shot Effect Type: -1~n

Changes the muzzle flash and sound effect when firing weapons such as Vulcans and Bullets.
This setting can also be used for the Player Bullet Effector, but player bullets have their own effects and do not use enemy bullet effects.

Extra Flags & Options
Extra Flags & Options
- Change auto destroy flag
Auto destroy flag: 0~1 (Default: 1)
&1: If core part, part will self-destruct when it has no attached destructible parts

- Change create flag when destroyed
Create flag when destroyed: 0~ (Default: 0)
0: Use default setting
1: Produces nothing
&2: Creates item
&4: Small counterattack
&8: Medium counterattack
&16: Large counterattack
&32: Extra-large counterattack

- Change hit check flag
Hit check flag: 0~3 (Default: 1)
&1: Enable collision detection
&2: Don't remove collision when part is destroyed

- Change hit check type
Hit check type: 0~3 (Default: 3)
&1: Part collides with player
&2: Part collides with player bullets

- Change draw option
Draw option: 0~3 (Default: 3)
&1: Draw phasing effect when appearing and hiding
&2: Draw phasing effect when switching between normal and shadow

- Change Target marker flag
Target marker flag: 0~2 (Default: 0)
1: Red marker
2: Yellow marker

- Change destroy effect flag
Destroy effect flag: 0~31 (Default: 31)
&1: Sound effect
&2: Explosion effect
&4: Spark effect
&8: Flash effect
&16: Shock wave effect

Sets the flags for effects produced when parts are destroyed.
This also applies to explosions produced by the Explosion Effector.

- Change destroy effect size
Destroy effect size: 0~500% (Default: 100)

This does not apply to explosions produced by the Explosion Effector.
Use extra parameters to control the effector's effect size.

- Change destroy delay
Destroy delay: 0 or 1~65535 (Default: 0)
0: Default value for part

Sets the delay for causing chain explosions on child parts.
This delay also applies to invisible parts, which are normally deleted immediately without any delay.

- Change kill count flag
Kill count flag: 0~15 (Default for core: 5) (Default for parts: 4)
&1: When destroyed, increase kill count (Core)
&2: When suicided or deleted, increase kill count (Core)
&4: When destroyed, increase kill count (Parts)
&8: When suicided or deleted, increase kill count (Parts)

Kill count updates automatically when parts are destroyed.
Player Status
Player Status
- Change the stun value of player
Value A - Stun value: 0~65535

Stun reduces player energy regeneration and disables boosting.
The stun value affects the duration of the stun, with higher values lasting longer.
Setting this to 0 will cancel the stun.

- Change the bind value of player
Value A - Bind counter: 0~65535 frames
Value B - Bind power (Move): 0~100%
Value C - Bind power (Turn): 0~100%

Bind reduces the player's movement and rotation speed.

The bind counter specifies the duration of the bind in frames.
Bind power (Move) sets the move speed reduction. If set to 100, the player will be unable to move.
Bind power (Move) sets the rotation speed reduction. If set to 100, the player will be unable to aim.

- Change the restraint value of player
Value A - Restraint counter: 0~65535 frames
Value B - Restraint power: 0~1

Restraint negates the effects of the player's equipment.
It does not negate their ability to press the attack button, only the effects of it.

The restraint counter sets the duration of the restraint in frames.
If restraint power is set to 1, it will apply to all equipment.

- Change the position of player
Option: 0~1

Sets the player's position to the position of the target part.
Option has the following flags:
&1: Apply a visual effect

- Change the energy of player
Modifies the player's energy.

Option specifies how to treat the value.
None: Set energy to that value.
Percent: Set energy to that percentage of maximum.
Permil: Set energy to that many thousandths of maximum.

- Change the energy stock of player
Modifies the player's energy stock.

Option specifies how to treat the value.
None: Set energy stock to that value.
Percent: Set energy stock to that percentage of maximum.
Permil: Set energy stock to that many thousandths of maximum.
Parts search
Parts search
- Parts search setting
(BP:0,0) Starts searching from coordinates (0, 0).
(BP:Player) Starts searching from the player's coordinates.
(BP:Parts) Starts searching from the coordinates of any part.

Search Options
0: Search parts from this core, use own SearchIndex, sort results by index order, include partial matches
&1: Search all parts
&2: Use core's SearchIndex
&4: Return zero when search index is out of range
&16: Sort results in ascending order
&32: Sort results in descending order
&128: Only include perfect matches

Search target
0: No specific target
&1: Cores
&2: Non-Cores
&4: Health bars
&8: Target marker A
&16: Target marker B
&32: Parts with durability (not invincible, max durability is positive)
&64: Parts without durability (invincible or max durability is negative)
&128: Parts with collision detection

- Parts search (Distance)
This event searches for parts to set as a pair part.
Before using it, you must set parts search settings on this part.

Searches based on distance from the part's BP position.
Range is measured in pixels. Input -1 to have no maximum range.

- Parts search (Direction)
This event searches for parts to set as a pair part.
Before using it, you must set parts search settings on this part.

The search sweeps in an angle, starting from the part's BP angle.
The range is measured in an angle from 0~2047 on both sides of the BP angle.
Set to -1 to search all the way around the part.
